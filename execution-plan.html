LinkNode – Execution Plan

Multi-tier supply chain digital twin roadmap with goals, frontend, and backend deliverables for each phase.

1️⃣

Data Foundation & Supplier Network

Phase 1

Goal: Supply chain ni digital copy banavvi (who depends on whom).

Frontend (HTML / CSS / JS)

  • Create base layout with sidebar plus main screen.
  • Build company registration screen (Anchor company).
  • Add form to enter suppliers under company.
  • Allow adding suppliers under suppliers (nested inputs).
  • Create dropdown to select role (Main company / Supplier).
  • Validate city name and location inputs.
  • Show confirmation screen after saving data.
  • Add loader animation while data saves.
  • Build supplier list page with search option.
  • Save login state in browser storage.
  • Create one common JS file to talk to backend APIs.

Backend (Python / Flask / MongoDB)

  • Create MongoDB database and collections.
  • Design supplier data model with parent link.
  • Create API to save company and supplier data.
  • Auto-generate unique supplier IDs.
  • Store parent-child relationships.
  • Create API to fetch supplier tree by company.
  • Add basic login security.
  • Track data changes (created / updated).
  • Add sample Tier-3 suppliers using seed data.
  • Handle API errors safely.
2️⃣

Network Map & Visual Flow

Phase 2

Goal: Supply chain ne map par visually batavvi.

Frontend (HTML / CSS / JS)

  • Integrate interactive world map.
  • Place suppliers as dots using city location.
  • Show supplier details on click.
  • Draw connecting lines between suppliers.
  • Use colors for status (Safe / Risk).
  • Add region focus buttons (India, Europe, USA).
  • Toggle connections on/off.
  • Switch map view modes.
  • Auto-focus map where most suppliers exist.
  • Highlight main company visually.

Backend (Python / Flask)

  • Create API to send supplier location data.
  • Convert data into map-friendly format.
  • Group suppliers by level.
  • Validate stored location data.
  • Send only required map data.
  • Cache map data for speed.
  • Filter suppliers by visible area.
  • Serve icons and marker assets.
  • Provide quick info API for map popups.
  • Log map access activity.
3️⃣

Sentinel – Risk News Detection

Phase 3

Goal: Real-world problems ne system sathe jodvu.

Frontend (HTML / CSS / JS)

  • Show live news alert bar.
  • Display risk cards in sidebar.
  • Show “time ago” format for alerts.
  • Blink indicator for new risk.
  • Open full news detail popup.
  • Allow dismissing alerts.
  • Use icons for different risk types.
  • Filter alerts by severity.
  • Mark alerts as read.
  • Optional sound alert for major risks.

Backend (Python Worker)

  • Connect to news API securely.
  • Scan news for risk keywords.
  • Identify affected city from news.
  • Save risk events in database.
  • Mark suppliers in that city as “At Risk”.
  • Limit API calls.
  • Keep last 30 days of history.
  • Notify main server when risk detected.
  • Link news to suppliers automatically.
  • Store severity level.
4️⃣

Risk Impact & Delay Calculation

Phase 4

Goal: Ek problem thi ketli delay thase te batavvu.

Frontend (HTML / CSS / JS)

  • Turn supply lines red when risk hits.
  • Show list of affected suppliers.
  • Display total delay days clearly.
  • Show impacted materials/products.
  • Animate flow of risk visually.
  • Show delay chart.
  • Auto-zoom to problem area.
  • Highlight weakest connection.
  • Hover effects for clarity.
  • Show overall risk meter.

Backend (Python / Logic)

  • Fetch all at-risk suppliers.
  • Calculate delay using buffer days.
  • Track direct & indirect impact.
  • Store impact score.
  • Find weakest supplier link.
  • Save impact history.
  • Provide API for impact summary.
  • Optimize calculations.
  • Prepare future prediction base.
  • Trigger notifications.
