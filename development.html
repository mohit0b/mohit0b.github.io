LinkNode Roadmap

Total Development Plan with Upgradation

A phased delivery roadmap ensuring LinkNode evolves from multi-tier visibility and predictive intelligence into a resilient, compliant digital twin platform.

1

Knowledge Graph & DB Architecture

Phase 1

Goal: Build the digital twin of the supply chain.

Frontend (HTML/CSS/JS)

  • Design the MainLayout shell with a fixed sidebar.
  • Create the onboarding form for Anchor registration.
  • Build a dynamic input field to add multiple sub-suppliers.
  • Style the role selector UI for Anchor versus Supplier.
  • Add JavaScript validation for GPS coordinate inputs.
  • Design a success state modal for completed registration.
  • Create a loading skeleton for the supplier directory.
  • Build a search bar to filter the supplier list by name.
  • Use local storage to keep the logged-in session active.
  • Write an Axios service for all backend communication.

Backend (Node.js/MongoDB)

  • Initialize the MongoDB Atlas cluster.
  • Design the SupplierSchema with parent_ids[].
  • Implement the POST /api/onboard endpoint.
  • Create logic to generate unique company IDs.
  • Set up the RelationshipSchema to track linkages.
  • Build the GET /api/suppliers/:id recursive fetch.
  • Implement bcrypt for secure user password storage.
  • Configure Mongoose middleware to log data changes.
  • Create a seed script to populate Tier-3 dummy data.
  • Add error-handling middleware for API safety.
2

Network Visualization (Mapbox GL JS)

Phase 2

Goal: Turn data into a 3D interactive globe.

Frontend (HTML/CSS/JS)

  • Embed Mapbox GL JS into a full-screen container.
  • Map JSON supplier data into Mapbox markers.
  • Create custom HTML/CSS popups for marker clicks.
  • Draw SVG arcs between nodes to show trade routes.
  • Style node colors: Anchor (blue) and Tier-1 (white).
  • Add “Zoom to Region” controls for Asia and Europe.
  • Provide a toggle to show or hide connection lines.
  • Offer a satellite view versus street view switcher.
  • Calculate map center based on supplier density.
  • Add a CSS pulse animation to highlight the Anchor node.

Backend (Node.js/Express)

  • Create the GET /api/graph/geo endpoint.
  • Convert raw database data into GeoJSON format.
  • Group suppliers by tier level for efficient rendering.
  • Add coordinate validation (latitude/longitude) in the database.
  • Optimize queries to return only visible nodes.
  • Implement a caching layer for map responses.
  • Add a bounding box filter to the geo endpoint.
  • Serve static supplier icons for map markers.
  • Expose an endpoint for supplier quick-info tooltips.
  • Log every map-fetch request for audit trails.
3

The Sentinel (Currents API Intelligence)

Phase 3

Goal: Connect the platform to real-world disruption signals.

Frontend (HTML/CSS/JS)

  • Build a live news ticker in the header.
  • Design a risk card component for the sidebar.
  • Format API timestamps into relative “X mins ago.”
  • Add a blinking red indicator for new alerts.
  • Create a news detail modal for full article reading.
  • Include a dismiss alert button using JS filters.
  • Design category icons for each risk type.
  • Build a filter UI to sort news by severity level.
  • Add a “Mark as Read” experience for notifications.
  • Trigger audio alerts when critical news hits.

Backend (Python/Node.js)

  • Set up a Python background worker.
  • Integrate the Currents API using secured keys.
  • Create a keyword filtering array (strike, flood, etc.).
  • Implement NLP sentiment analysis on headlines.
  • Extract locations from news items for geo-tagging.
  • Build the POST /api/risks ingestion logic.
  • Rate-limit Currents API calls to avoid throttling.
  • Flag linked suppliers as affected when relevant.
  • Set up webhooks to notify the Node.js server.
  • Store a 30-day history of risk events in MongoDB.
4

Risk Propagation & Impact Logic

Phase 4

Goal: Reveal how delays ripple through the network.

Frontend (HTML/CSS/JS)

  • Turn Mapbox connection lines red when a child is at risk.
  • Build an impact path list showing the delay chain.
  • Design a delay counter UI (for example, “15 days delay”).
  • List affected products with badge styling.
  • Animate pulses along lines connecting at-risk nodes.
  • Render a risk summary bar chart using Chart.js.
  • Auto-zoom the map to the source risk node.
  • Style line weights based on trade volume.
  • Add hover effects to highlight specific sub-chains.
  • Introduce a global risk meter gauge on the dashboard.

Backend (Node.js/Logic)

  • Implement MongoDB $graphLookup for recursion.
  • Calculate direct versus indirect risk per node.
  • Update the impact_score in the database.
  • Compute buffer logic (Delay - Inventory = Loss).
  • Identify the critical path (weakest supply link).
  • Trigger automated risk propagation functions.
  • Store historical impact records for future prediction.
  • Expose an endpoint for downstream victims list.
  • Optimize graph traversal for performance.
  • Dispatch email/SMS alerts via a notification service.
5

Command Center & Analytics Dashboard

Phase 5

Goal: Deliver an executive-grade visibility dashboard.

Frontend (HTML/CSS/JS)

  • Create a “Total Tiers Managed” counter widget.
  • Integrate a pie chart for risk distribution.
  • Design a regional health heatmap table.
  • Apply dark-mode glassmorphism styling.
  • Add a date range picker for historical analysis.
  • Build supplier cards with mini sparkline graphs.
  • Implement map search for specific nodes.
  • Design a compliance overview progress bar.
  • Smooth transitions between map and graph views.
  • Create a download PDF report button.

Backend (Node.js/Analytics)

  • Aggregate supplier health from MongoDB.
  • Provide an endpoint for the top five riskiest suppliers.
  • Calculate a supply chain resiliency score (0–100).
  • Add pagination for large supplier datasets.
  • Expose summary stats for the dashboard header.
  • Track average response time to risks.
  • Schedule a daily snapshot cron job.
  • Group analytics by material type.
  • Secure analytics endpoints with admin roles.
  • Build data export utilities for JSON and CSV.
6

Alternative Sourcing & Mitigation

Phase 6

Goal: Present intelligent plan-B pathways.

Frontend (HTML/CSS/JS)

  • Design a plan-B recommendation modal.
  • Add compare UI for current versus alternative suppliers.
  • Include a request quote button for new vendors.
  • Style trust score stars for supplier ratings.
  • Provide a switch supplier slider for anchors.
  • Show cost impact indicators for each switch.
  • Highlight the new logistics route on the map.
  • Add verified badges for compliant alternatives.
  • Show alerts when no alternatives are available.
  • Offer an undo switch experience within 24 hours.

Backend (Node.js/Search)

  • Create the discovery pool database of suppliers.
  • Build the material match ranking algorithm.
  • Implement a geographic safe-zone filter.
  • Rank alternatives by trust score and distance.
  • Execute pivot logic to update graph edges.
  • Add compliance checks into alternative filters.
  • Expose a supplier recommendation API endpoint.
  • Log all avoidance actions for reporting.
  • Allow manual sourcing entries for anchors.
  • Adjust inventory buffers based on new routes.
7

Digital Vault & Compliance Tracking

Phase 7

Goal: Guarantee legal, ethical, and transparent sourcing.

Frontend (HTML/CSS/JS)

  • Design a drag-and-drop file upload zone.
  • Create a compliance status dashboard with color states.
  • Build an expiry alert countdown for certificates.
  • Style a document viewer modal with preview.
  • Filter documents by category (ISO, ESG, etc.).
  • Add a verified checkmark animation.
  • List missing documents per supplier.
  • Offer bulk download for audit preparation.
  • Rank suppliers by document completeness.
  • Provide a digital signature placeholder UI.

Backend (Node.js/Storage)

  • Implement Multer for secure PDF handling.
  • Set up a documents collection in MongoDB.
  • Run cron jobs to detect expiring certificates.
  • Hash files to ensure document integrity.
  • Configure AWS S3 or local storage integrations.
  • Provide GET /api/documents/:supplierId access.
  • Trigger auto-reminder emails for lapsing documents.
  • Record upload timestamps and uploader identities.
  • Auto-disable nodes that fall out of compliance.
  • Enforce file size and type restrictions.
8

Deployment & Simulation Mode

Phase 8

Goal: Prepare a high-impact demonstration environment.

Frontend (HTML/CSS/JS)

  • Build a hidden simulation control panel.
  • Add buttons such as “Trigger Cyclone” or “Port Strike.”
  • Create a demo speed slider to fast-forward time.
  • Develop a tour guide overlay for exhibitions.
  • Design a reset system control to clear mock risks.
  • Badge the interface as production ready.
  • Include a contact developer popover.
  • Auto-play a scenario video during demos.
  • Offer a mobile preview link for reviewers.
  • Finalize CSS for edge-to-edge immersive layouts.

Backend (Node.js/DevOps)

  • Deploy the API services to Render or Railway.
  • Run the Python Sentinel as a managed worker.
  • Securely manage environment variables and keys.
  • Expose manual disaster override APIs.
  • Seed a dedicated demo database.
  • Configure CORS for the production frontend domain.
  • Enable logging and monitoring (for example, Morgan).
  • Optimize MongoDB indexes for final scenarios.
  • Provide a health check endpoint.
  • Complete a security audit on JWT tokens.
